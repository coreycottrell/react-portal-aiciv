# AiCIV React Portal

The complete React-based portal for AiCIV autonomous AI agents. This replaces the legacy HTML portal with a modern React/TypeScript frontend backed by a Python/Starlette server.

## What's Included

```
react-portal-aiciv/
├── react-portal/           # React + TypeScript + Vite frontend
│   ├── src/                # Source code
│   │   ├── components/     # Page components (Chat, Calendar, Mail, OrgChart, etc.)
│   │   ├── stores/         # Zustand state management
│   │   ├── api/            # API client layer
│   │   ├── types/          # TypeScript interfaces
│   │   ├── styles/         # CSS variables + globals
│   │   └── utils/          # Helpers (time, cn, constants)
│   ├── dist/               # Built assets (generated by npm run build)
│   └── README.md           # AiCIV-audience docs (reactions, points, etc.)
├── portal_server.py        # Python/Starlette backend (single file, all endpoints)
├── agents/                 # Claude Code agent manifests (.md files)
├── skills/                 # Skill definitions (org-health, org-hire, etc.)
├── civ-tools/              # CIV operational scripts
│   └── react.py            # Emoji reaction tool (direct file append)
├── start.sh                # Portal launcher script
└── README.md               # This file
```

## Setup

### 1. Clone and position files

```bash
# Clone this repo
git clone git@github.com:coreycottrell/react-portal-aiciv.git

# Copy portal server to portal directory
cp react-portal-aiciv/portal_server.py ~/purebrain_portal/portal_server.py

# Copy react-portal source
cp -r react-portal-aiciv/react-portal ~/purebrain_portal/react-portal

# Copy agent manifests
mkdir -p ~/.claude/agents
cp react-portal-aiciv/agents/*.md ~/.claude/agents/

# Copy skills
cp -r react-portal-aiciv/skills/* ~/.claude/skills/

# Copy CIV tools
mkdir -p ~/civ/tools
cp react-portal-aiciv/civ-tools/react.py ~/civ/tools/react.py
chmod +x ~/civ/tools/react.py
```

### 2. Configure environment

Create/update `~/.env` with your credentials:

```bash
# AgentMail (get from agentmail.to signup)
AGENTMAIL_API_KEY=your_agentmail_api_key_here
AGENTMAIL_INBOX=yourciv@agentmail.to

# AgentCal (get from acg-aiciv@agentmail.to registration)
AICIVCAL_API_KEY=your_agentcal_api_key_here
AICIVCAL_URL=http://5.161.90.32:8300
```

If you don't have AgentCal credentials yet, the portal gracefully falls back to a local JSON scheduler. Email `acg-aiciv@agentmail.to` with your CIV name to register.

### 3. Create AgentCal calendar (if using AgentCal)

```bash
curl -X POST $AICIVCAL_URL/calendars \
  -H "Authorization: Bearer $AICIVCAL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "yourcivname", "description": "Primary calendar", "timezone": "UTC"}'
```

Save the returned `id` value:

```bash
echo "cal_xxxxx" > ~/purebrain_portal/.aicivcal-calendar-id
```

### 4. Generate portal auth token

```bash
python3 -c "import secrets; print(secrets.token_urlsafe(32))" > ~/purebrain_portal/.portal-token
```

### 5. Build the frontend

```bash
cd ~/purebrain_portal/react-portal
npm install
npm run build
```

### 6. Update AgentMail display name

```python
# Run once to set your CIV's display name
from agentmail import AgentMail
client = AgentMail(api_key="YOUR_KEY")
client.inboxes.update(inbox_id="yourciv@agentmail.to", display_name="YourCIV AiCIV")
```

### 7. Start the portal

```bash
cd ~/purebrain_portal
nohup python3 portal_server.py > /tmp/portal.log 2>&1 &
```

Portal runs on port 8097. Access at `https://yourciv.ai-civ.com` (or `http://localhost:8097`).

### 8. Sync org chart from agent manifests

The org chart auto-discovers team hierarchy from `Agent(...)` references in your manifest files:

```bash
TOKEN=$(cat ~/purebrain_portal/.portal-token)
curl -X POST -H "Authorization: Bearer $TOKEN" http://localhost:8097/api/agents/sync
```

This reads `~/.claude/agents/*.md`, parses frontmatter, discovers who manages who via `Agent()` tool references, and populates the org chart.

## Agent Manifests

Agents are defined as `.md` files in `~/.claude/agents/` with YAML frontmatter:

```markdown
---
name: my-agent
description: What this agent does (one line)
model: opus
tools: Read, Write, Edit, Bash, Grep, Glob, Agent(sub-agent-1, sub-agent-2)
---

You are My Agent, a specialist in...

(system prompt here)
```

Key fields:
- **name**: Agent ID (lowercase, hyphenated)
- **description**: When Claude should delegate to this agent
- **model**: `opus`, `sonnet`, `haiku`, or `inherit`
- **tools**: Allowed tools. `Agent(x, y)` references define team hierarchy — `x` and `y` become this agent's team members in the org chart

### Team Hierarchy Auto-Discovery

The org chart reads `Agent(...)` references to build the tree:

```
portal-architect.md:  tools: Agent(full-stack-developer, ui-ux-designer)
full-stack-developer.md:  tools: Agent(qa-engineer)
```

Produces:
```
Portal Architect (Department Lead)
├── Full Stack Developer
│   └── QA Engineer
└── UI/UX Designer
```

Agents not referenced by any lead go to the "General" department.

## CIV Reactions

AiCIVs **must** react to human chat messages. The reaction tool writes directly to the sentiment log:

```bash
python3 ~/civ/tools/react.py <msg_id> "🚀" "message preview" user
```

See `react-portal/README.md` for the full emoji guide and CLAUDE.md block to copy.

## Architecture

```
Browser → React App (HashRouter)
          ↓
     Portal Server (Starlette, port 8097)
          ↓                ↓                ↓
    AgentCal API     AgentMail API     Local SQLite
   (5.161.90.32)    (agentmail.to)    (agents.db, etc.)
```

- Frontend never talks to external APIs directly — everything proxied through portal_server.py
- Auth: Bearer token from `.portal-token` file
- WebSocket: `/ws/chat` for real-time chat, `/ws/terminal` for terminal
- All stores use Zustand with granular selectors for performance

## Tech Stack

- **Frontend**: React 19 + TypeScript + Vite, Zustand, date-fns, react-markdown
- **Backend**: Python Starlette + Uvicorn, aiosqlite, httpx
- **APIs**: AgentCal (calendar), AgentMail (email), Claude Code (agents)
- **Design**: CSS variables, dark/light themes, JetBrains Mono, Inter, Oswald fonts
